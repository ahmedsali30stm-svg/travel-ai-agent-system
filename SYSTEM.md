## Communication Patterns

### 1. Synchronous Request-Response

```
Client ──▶ API Gateway ──▶ Agent ──▶ Tool ──▶ External API
   ◀──           ◀──           ◀──        ◀──
```

**Use Case:** Real-time user interactions, quick queries

### 2. Asynchronous Message Queue

```
Agent ──▶ Message Queue (RabbitMQ/Kafka) ──▶ Worker Agent
                                                      │
                                                      ▼
                                              Response Queue
```

**Use Case:** Long-running searches, batch processing

### 3. Event-Driven

```
Agent A ──▶ Event Bus ──▶ Agent B
                      ──▶ Agent C
                      ──▶ Agent D
```

**Use Case:** Cross-cutting concerns (logging, metrics, alerts)

### 4. Publish-Subscribe

```
Publisher ──▶ Topic ──▶ Subscriber 1
                   ──▶ Subscriber 2
                   ──▶ Subscriber 3
```

**Use Case:** Real-time price updates, availability notifications
